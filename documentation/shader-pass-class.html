ShaderPass 类

源码

基本使用

Shader passes 提供了向一组平面或整个场景添加后期处理的可能性。可以将其想象为一个额外的、大小与WebGL画布相同的平面，它将一组平面渲染到一个纹理中并进行绘制。这样你就可以在片元着色器中操作那个纹理。 Shader passes offers the possibility to add post-processing to either a bunch of planes or your whole scene. Think of it as an additional plane that would have the size of your WebGL canvas and that would render a set of planes into a texture and draw it. You'd then be able to manipulate that texture in your fragment shader.

就像 Plane类 一样，ShaderPass 类也继承自 Mesh 和 DOMMesh 类，并且它进一步扩展了 DOMMesh 类的功能，通过管理帧缓冲对象（frame buffer object）的创建和渲染纹理（render texture）的生成。 Like the Plane class it inherits from the Mesh and DOMMesh classes.
ShaderPass extends the DOMMesh class by handling the creation of a frame buffer object and the creation of a render texture.

需要注意的是，渲染目标（以及因此而来的Shader Passes）会禁用WebGL上下文的默认抗锯齿功能。如果你使用它们，在初始化上下文时，应将Curtains属性中的antialias设置为false。
之后，你可能希望添加一个额外的抗锯齿处理步骤，可以通过内置的 FXAAPass 类实现，或者采用你自己的FXAA或MSAA实现方法。
有关如何添加FXAAPass的一个示例，请参考 Post processing scrolling wheel with custom transform origin 这个例子的源代码或在线演示。 Please note that render targets (and therefore shader passes) disable the WebGL context default antialiasing. If you use them, you should set the antialias Curtains property to false when initiating your context.
You might then want to add an extra antialiasing pass, by using the built-in FXAAPass class or your own FXAA or MSAA implementation.
See the Post processing scrolling wheel with custom transform origin for an example of how to add a FXAAPass.

创建

为了创建一个将应用于整个场景的着色器通道（Shader Pass），请使用其构造函数：

// “canvas” 是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: "canvas"
});
// 使用Curtains对象创建一个新的着色器通道
const shaderPass = new ShaderPass(curtains);

参数

上述代码将创建一个基础的着色器通道，但你可能无法直接对其进行太多操作。
可以在 new ShaderPass() 中传递第二个参数，传递所有需要的选项：

// “canvas” 是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: "canvas"
});
const shaderPassParams = {
vertexShaderID: "my-shader-pass-vs", // 顶点着色器script标签的 id
fragmentShaderID: "my-shader-pass-fs", // 片段着色器script标签的 ID
uniforms: { // uniforms（统一变量）是允许你与着色器通道进行交互的关键
time: {
name: "uTime", // 传递给着色器的uniform变量名称
type: "1f", // 这意味着我们的uniform变量是一个浮点数
value: 0, // uniform变量的初始值
},
},
};
// 使用Curtains对象和上述参数创建一个新的着色器通道
var shaderPass = new ShaderPass(curtains, shaderPassParams);

着色器

着色器通道使用的顶点着色器和片元着色器与 平面（Plane）对象 稍有不同。

顶点着色器（Vertex shader）

由于着色器通道并不包含投影矩阵和模型视图矩阵，因此默认的顶点着色器极其简单（并且是可选的）。
如果你在着色器通道中添加了新的纹理，请务必在这个顶点着色器中处理该纹理对应的变换矩阵。 Since shader passes does not have projection and model view matrices, the default vertex shader is extremely simple (and optional).
If you add a new texture to your shader pass, do not forget to handle its matrix in this vertex shader.

precision mediump float;
// 这些是库中设定的必要属性
attribute vec3 aVertexPosition;
attribute vec2 aTextureCoord;
// 将顶点坐标和纹理坐标传递给片段着色器
varying vec3 vVertexPosition;
varying vec2 vTextureCoord;
void main() {
gl_Position = vec4(aVertexPosition, 1.0);
// 设置变量
// 在片段着色器中，使用aTextureCoord属性作为纹理坐标
vTextureCoord = aTextureCoord;
vVertexPosition = aVertexPosition;
}

片段着色器（Fragment shader）

ShaderPass 对象在创建时会自动生成一个纹理，用于存储你的着色器通道帧缓冲区（FBO）内容。若要显示着色器通道的内容，你需要展示这个纹理。如果你想应用任何后期处理效果，这将是通过操作该纹理实现的。
这个纹理采样器的名字是 uRenderTexture。 The ShaderPass object automatically creates a texture holding your shader pass FBO content upon creation. You will have to display that texture to display your shader pass content. If you want to apply any post processing effect, this will be by manipulating this texture.
This texture sampler name is uRenderTexture.

precision mediump float;
// 获取变量
varying vec3 vVertexPosition;
varying vec2 vTextureCoord;
// 我们的渲染纹理
uniform sampler2D uRenderTexture;
void main() {
// 显示我们的渲染纹理，其中包含了着色器通道帧缓冲对象的内容
gl_FragColor = texture2D(uRenderTexture, vTextureCoord);
}

完整参数列表

  • curtains Curtains class object Curtains 示例对象
  • params object，可选 包含plane参数的对象：
    • vertexShader string，可选 作为字符串提供的顶点着色器代码。需要注意代码中的换行符，因为它可能导致 JavaScript 错误。如果不指定，则会查找 vertexShaderID 参数。
    • vertexShaderID string，可选 顶点着色器ID。如果没有指定，则会使用默认的顶点着色器。
    • fragmentShader string，可选 作为字符串提供的片段着色器代码。同样要注意换行符以免引发JavaScript错误。如果不指定此参数，则会查找 fragmentShaderID 参数。
    • fragmentShaderID string，可选 片段着色器ID。若未指定任何内容，则将使用一个默认的片段着色器，它会不作修改地绘制场景。
    • renderOrder int，可选 定义着色器通道被绘制的顺序。参见 renderOrder 属性。默认值为0。
    • depthTest bool，可选 是否为该着色器通道启用或禁用深度测试功能。默认值为 true。
    • depth bool，可选 设定着色器通道渲染目标是否应使用深度缓冲区（参见RenderTarget类对象）。默认值为 false。
    • clear bool，可选 决定在绘制之前是否清除着色器通道渲染目标的内容（参见RenderTarget类对象）。默认值为 true。
    • renderTarget RenderTarget class object，可选 可以提供一个已经存在的渲染目标来使用。默认值为 null。
    • texturesOptions object，可选 用于着色器通道纹理的默认选项，参照Texture类参数。
    • crossOrigin string，可选 定义在加载图片时（如果有需要的话）使用的跨域处理方式，默认为 "anonymous"。
    • uniforms object，可选 将要传递给着色器的统一变量。如果未指定任何统一变量，则与着色器通道没有交互。
      • name string 在您的着色器代码中使用的统一变量名称。
      • type string 统一变量的数据类型。可参考 WebGL Fundamentals 页面获取所有可用的统一变量类型信息，包括但不限于 float、int、vec2、vec3、mat4 等。
      • value float, integer, Vec2, Vec3, Mat4 or array of floats or integers depending on your uniform type 统一变量的具体值，其数据类型应与声明的 uniform type 相匹配。

属性

  • canvases (array): v3.0

    包含通过加载方法加载到着色器通道的所有canvas元素。

  • crossOrigin (string): v3.0

    指定加载媒体资源时使用的跨域策略。

  • cullFace (string): v7.0只读

    决定着色器通道中哪一面（正面或背面）不被绘制，默认仅剔除背面。

  • gl (WebGL context) v7.0只读

    提供当前WebGL上下文。

  • htmlElement (HTML element): v3.0

    创建着色器通道所用的HTML元素，即 Curtains container。

  • images (array): v3.0

    包含通过加载方法加载到着色器通道的所有图像资源的数组

  • index (integer): v3.0 只读

    表示着色器通道在Curtains对象的shaderPasses数组中的索引位置。

  • loader (PlaneTextureLoader class object): v7.0只读

    着色器通道内部的纹理加载器。

  • renderer (Renderer class object): v7.0只读

    指向由Curtains对象创建的Renderer实例。

  • renderOrder (int): v7.3只读

    此属性为着色器通道的渲染顺序值，它决定了各着色器通道被绘制的具体顺序。
    若一个着色器通道的 renderOrder 值较高，则其会在 renderOrder 值较低的着色器通道之后进行绘制。
    若要更改此值，请使用 setRenderOrder() 方法。默认情况下该值为0。
    了解更多关于场景中对象绘制顺序的信息，请参阅 Scene rendering order 文档。

  • target (RenderTarget class object): v5.0

    用于渲染着色器通道的渲染目标。

  • textures (array): v3.0

    包含着色器通道已创建的所有Textures对象的数组

  • type (string): v7.0只读

    表示类对象类型，此处是"ShaderPass"。

  • uniforms (object): v3.0updated in v7.0

    该对象包含了你作为参数传递的所有 uniforms（统一变量）。
    你可以通过修改其 value 属性来更新你的 uniform 值。
    自从 v7.0 版本开始， 你可以直接使用 Vec2， Vec3， Mat4 以及 Quat 对象作为 uniform 的值 ，而无需再使用数组。

  • userData (object): v4.2

    一个空的对象，用于在你的着色器通道中存储任何额外的数据或自定义属性。

  • uuid (string): v6.0只读

    着色器通道的唯一标识符。

  • videos (array): v3.0

    一个数组，包含所有通过加载方法加载到着色器通道中的视频资源。

方法

  • addTexture(texture): v7.0

    此函数接受一个已存在的纹理，并将其添加到着色器通道中。

    • texture Texture class object 要添加到此着色器通道的纹理。等同于 Texture 类的 addParent() 方法。

  • createTexture(params): v2.0updated in v5.1

    此函数将创建一个新的 Texture，并将其添加至 ShaderPass 中。
    新创建的纹理将会被返回。

    • params object，可选 包含纹理参数的对象：
      • sampler string，可选 用于设置着色器中采样器 uniform 的名称。也会影响纹理矩阵 uniform 的名称。如果不指定，默认将采样器命名为 "uSampler" 加上此纹理在着色器通道中的索引。
      • fromTexture Texture object，可选 已存在的纹理，用于复制到新创建的纹理中。类似于 Texture 类的 setFromTexture() 方法。

    返回值：如果成功，返回新创建的 Texture 对象。

  • getBoundingRect(): v3.0

    获取 container HTML 元素的边界矩形而不触发重新布局或重绘。请注意，这些值是相对于你的 Curtains pixelRatio 值的。

    返回值：包含 container HTML 元素宽度、高度、顶部和左侧位置的对象。

  • loadCanvas(canvasElement, textureOptions, successCallback): v3.0updated in v7.0

    此函数用于处理一个 HTML canvas 元素，基于该元素创建一个 Texture 对象，并将其加载至着色器通道。

    • canvasElement HTML canvas element 要加载到着色器通道中的 HTML canvas 元素。
    • textureOptions object，可选 应用到该纹理的默认选项。请参阅 Texture 类的 参数 部分获取详细信息。
    • successCallback function(texture)，可选 加载成功时执行的回调函数。
      • texture Texture class object 新创建的纹理对象。在此处您可以访问到纹理的 onSourceLoaded 和 onSourceUploaded 回调方法。

    示例： Text planes using canvas

  • loadCanvases(canvasElements, texturesOptions, successCallback): v3.0updated in v7.0

    • texturesOptions object，可选 应用到该纹理的默认选项。请参阅 Texture 类的 参数 部分获取详细信息。
    • successCallback function(texture)，可选 成功加载时执行的回调函数
      • texture Texture class object 新创建的纹理对象。在此处您可以访问到每个纹理的 onSourceLoaded 和 onSourceUploaded 回调方法。

  • mouseToPlaneCoords(xMousePosition, yMousePosition): v3.0

    此函数用于获取鼠标坐标相对于着色器通道裁剪空间的值。您可以将这些坐标发送到 uniforms，并与着色器通道进行交互。着色器通道中的坐标范围从左上角的 (-1, 1) 到右下角的 (1, -1)，这意味着沿 Y 轴方向的值是反转的。

    • xMousePosition float 鼠标事件的 clientX 值
    • yMousePosition float 鼠标事件的 clientY 值

  • playVideos(): v3.0

    这个函数会自动启动所有着色器通道中的视频播放。如果在用户交互动作之前调用它，在移动设备上可能无法正常工作。使用此方法确保所有关联的视频内容开始播放。

  • remove(): v7.0

    该函数将从Curtains对象及其渲染器中移除指定的着色器通道及其渲染目标，并删除其所有的纹理资源。同时，此操作还会更新所有其他着色器通道的索引顺序。

  • setRenderOrder(renderOrder): v7.3

    设置着色器通道的新 渲染顺序。
    着色器通道将会在具有更低 renderOrder 数值的着色器通道之后被绘制。允许使用负数作为 renderOrder。若省略 renderOrder 参数，可以重置着色器通道的 renderOrder 值。

    • renderOrder int 要使用的新的渲染顺序值，默认为0。

  • onAfterResize(callback): v3.0

    返回值：你的 ShaderPass 对象，这使得它可以进行链式调用。

  • onError(callback): v7.0

    当在实例化着色器通道时出现错误时，会调用此函数。这通常是因为着色器编译失败所导致的。

    • callback function 回调函数

    返回值：你的 ShaderPass 对象，这使得它可以进行链式调用。

  • onLoading(callback(texture)): v3.0

    每当着色器通道的源元素（无论是图像、画布还是视频）加载完毕并准备好显示时，此函数将被触发。在这里，您可以访问到新创建的纹理，这对于处理加载器或操作纹理非常有用。

    • callback function 需要执行的函数，新创建的纹理将会作为参数传递给该函数。

    返回值：你的 ShaderPass 对象，这使得它可以进行链式调用。

    示例： Post processing displacement effect

  • onReady(callback): v3.0

    当我们的着色器通道完全设置完毕并准备好进行绘制时，此函数将会被调用。

    • callback function 回调函数

    返回值：你的 ShaderPass 对象，这使得它可以进行链式调用。

    示例： Post processing displacement effect

  • onRender(callback): v3.0

    此函数会在每次调用requestAnimationFrame时，针对每个着色器通道触发执行。它的用途在于在每一次渲染循环中更新那些随时间变化的 uniforms（统一变量）。

    • callback function 回调函数

    返回值：你的 ShaderPass 对象，这使得它可以进行链式调用。

    示例： Post processing displacement effect， Post processing multiple passes
