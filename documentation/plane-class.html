Plane 类

源码

基本使用

Plane 类继承自 BasePlane 类，BasePlane 类主要负责设置 WebGL（缓冲区和属性）以及加载平面的部分源内容。
Plane 类通过添加多种属性和方法扩展了 BasePlane 类，以确保 WebGL 平面与关联的 HTML 元素保持同步。所有尺寸和位置处理均由两个矩阵完成，您需要在顶点着色器内部使用这两个矩阵：模型视图矩阵(model view matrix)和投影矩阵(projection matrix)。这样，Plane 类不仅集成了基本的 WebGL 设置功能，还为平面提供了更丰富的同步控制和渲染能力。 Plane class inherits from the BasePlane class, which is a class that just sets up the WebGL (buffers and attributes) and sources loading parts of a plane.
Plane extends the BasePlane class by adding various properties and methods to keep your WebGL plane in sync with its associated HTML element. All sizes and positions are handled by two matrices that you will have to use inside your vertex shader: the model view matrix and the projection matrix.

要创建一个平面（plane），您需要传递 curtains 对象以及一个必需的 HTML 元素。这个 HTML 元素将用于计算plane的大小和位置。 To create a plane, you will need to pass your curtains object and a mandatory HTML element. It will be used to calculate the size and position of your plane.

// “canvas” 是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: "canvas"
});
const planeElement = document.getElementById("my-plane");
const plane = new Plane(curtains, planeElement);

参数

上述代码将创建一个基础平面，但您可能无法对它进行更多操作。Plane 类有第三个参数，您可以通过该参数传递所有需要的选项：

// “canvas” 是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: "canvas"
});
const planeElement = document.getElementById("my-plane");
const params = {
vertexShaderID: "my-plane-vs", // 顶点着色器（Vertex Shader）脚本标签的ID
fragmentShaderID: "my-plane-fs", // 片段着色器（Fragment Shader）脚本标签的ID
uniforms: { // uniforms 是允许您与平面进行交互的关键
time: {
name: "uTime", // 将传递给着色器的uniform名称
type: "1f", // 这意味着我们的uniform变量是一个浮点数类型
value: 0, // uniform变量的初始值
},
},
};
// 使用上述参数创建我们的平面
const plane = new Plane(curtains, planeElement, params);

通过HTML数据属性传递着色器ID

您还可以通过plane HTML元素分别使用data-vs-id和data-fs-id属性来传递顶点着色器和片段着色器的ID：

<!-- div 用于创建我们的平面 -->
<div id="my-plane" data-vs-id="my-plane-vs" data-fs-id="my-plane-fs">
<!-- 这张图片将被用作我们平面的纹理 -->
<img src="path/to/my-image.jpg" crossorigin="" />
</div>

完整参数表

以下是必需参数和可用选项的完整列表：

  • curtains Curtains class object 使用 Curtains 类创建出来的实例对象
  • planeElement HTML element 用于计算plane的大小和位置的HTML元素
  • params object，可选 一个包含平面参数的对象：
    • vertexShader string，可选 您的顶点着色器代码，以字符串形式提供。请注意换行符的处理，因为不当的换行可能会导致JavaScript错误。如果不指定，则会查找vertexShaderID参数。
    • vertexShaderID string，可选 顶点着色器ID。如果省略，将会在plane HTML元素上查找data-vs-id数据属性。如果没有指定任何内容，则会使用默认的顶点着色器，并抛出一个警告信息。
    • fragmentShader string，可选 您的片段着色器代码，以字符串形式提供。请注意换行符的处理，因为不当的换行可能会导致JavaScript错误。如果不指定，则会查找fragmentShaderID参数。
    • fragmentShaderID string，可选 片段着色器ID。如果省略，则会在平面HTML元素上查找`data-fs-id`数据属性，并尝试从中获取片段着色器ID。如果没有明确指定任何片段着色器，系统将使用一个默认的片段着色器，该着色器仅绘制黑色像素，并且会抛出一个警告信息。
    • widthSegments integer，可选 平面沿X轴方向的细分程度，默认值为1。这个参数用于定义平面在宽度方向上的细分数量，细分越多意味着平面在该方向上的几何细节越丰富，视觉效果上会更加平滑。
    • heightSegments integer，可选 平面沿Y轴方向的细分程度，默认值为1。这个参数用于定义平面在高度方向上的细分数量，细分越多意味着平面在垂直方向上的几何细节越丰富，视觉效果上会更加平滑和精细。
    • renderOrder int，可选 定义平面对象的绘制顺序，默认值为0。参照renderOrder属性。 在三维场景中，当存在多个相互重叠或透明的对象时，renderOrder 属性可以帮助确定这些对象被渲染的先后次序。默认情况下，所有对象的 renderOrder 值为0，引擎会根据它们在场景中的深度（远近）进行排序渲染（即深度测试）。然而，如果设置了自定义的 renderOrder 值，那么对象将按照这个值从小到大进行排序渲染，数值越小，优先级越高，越先被渲染。这对于需要特定渲染顺序以实现特殊效果或者确保某些对象总是在其他对象之上或之下显示的场景特别有用。
    • depthTest bool，可选 决定是否启用深度测试，默认值为 true。深度测试是一种确定像素在场景中的前后关系的技术，以防止较远的对象错误地绘制在较近的对象前面。
    • transparent bool，可选 控制平面对象是否开启透明效果处理，默认值为 false。如果设置为 true，则该平面将在不使用透明度的其他平面之后进行渲染，以正确显示透明度叠加效果。
    • cullFace 'back' | 'front' | 'none'，可选 指定应该剔除哪个面。可能的值包括 "back"（背面剔除，只渲染可见面）、"front"（正面剔除，仅渲染背面）、或 "none"（不剔除任何面）。默认值为 "back"，这意味着在渲染时会剔除那些面向观察者背面的多边形。
    • alwaysDraw bool，可选 定义平面是否始终被绘制，即使它完全位于场景视锥体之外（即不在摄像机视野范围内）。默认值为 false，这意味着如果平面完全不可见，则不会对其进行渲染。
    • visible bool，可选 简单地控制平面是否可见和需要被绘制。默认值为 true
    • drawCheckMargins object，可选 在绘制检查计算中添加的附加边距（以像素为单位）。
      正值表示即使在画布外小于该值也会绘制平面，负值表示在画布内大于该值也会隐藏平面。
      如果您弄乱了顶点着色器中的顶点位置，这很有用。 Additional margins to add in the draw check calculations, in pixels.
      Positive value means the plane will be drawn even if outside the canvas by less than the value, negative value means the plane will be hidden if inside the canvas by more than the value.
      Useful if you're messing with the vertices positions in your vertex shader.
      • top float 当检查平面顶部相对于画布底部的位置时应用的边距。默认值为0。正值意味着即使平面在画布顶部之外但不超过该边距值时，仍会绘制平面；负值则表示即使平面在画布内但在画布顶部以内超过该绝对值时，将隐藏平面。 margin to apply when comparing the top side of the plane and the bottom side of the canvas (0 by default).
      • right float 当比较平面右边界和画布左边界时应用的边距，默认也为0。同样，正值使得即便平面超出画布右侧少许也会被绘制出来，而负值则会导致当平面比画布左侧稍微靠里时不再显示。 margin to apply when comparing the right side of the plane and the left side of the canvas (0 by default).
      • bottom float 应用于比较平面底部与画布顶部位置的边距，默认是0。其作用原理同上，不过这里是针对垂直方向上的底部边缘。 margin to apply when comparing the bottom side of the plane and the top side of the canvas (0 by default).
      • left float 在对比平面左边界与画布右边界时所用的边距，初始设置也是0。这里，正值让平面即使稍微超出画布左侧也能继续渲染，负值则会让平面在过于靠近画布右侧内部时不再显现。 margin to apply when comparing the left side of the plane and the right side of the canvas (0 by default).
    • watchScroll bool，可选 是否在用户滚动时自动更新平面的位置（默认为 false）。
    • autoloadSources bool，可选 是否在初始化时自动加载资源（默认为 true）。
    • texturesOptions object，可选 用于平面纹理的默认选项。请参阅 Texture 类的参数。
    • crossOrigin string，可选 定义（如有需要）加载图片时采用的跨域处理方式，默认为 "anonymous"。
    • fov integer，可选 透视视场角度（默认为 50）。
    • uniforms object，可选 将传递给着色器的 uniform（如果不指定任何 uniform，则不会与平面进行任何交互）。
      • name string 在您的着色器中使用的 uniform 名称
      • type string uniform 的数据类型（请参考 相关文档）
      • value float | integer | Vec2 | Vec3 | Mat4 | (float | integer)[] uniform 的具体值

属性

  • alwaysDraw (boolean): v2.0

    该属性决定WebGL渲染器是否应始终绘制平面，或者仅在平面位于canvas视口内时才进行绘制。此属性用于性能优化。

    查看 Plane properties and transformations cheat sheet 示例，了解如何实际应用此设置。

  • autoloadSources (boolean): v2.0只读

    该属性表示在平面初始化时，是否应自动加载平面的源（如图片、视频以及作为HTML子元素的canvas）并据此创建相应的纹理。

  • camera (Camera class object): v7.0只读

    该属性代表用于计算平面投影矩阵的透视相机。这意味着，当你需要了解或间接操作与该平面相关的摄影机设置时，可以通过此属性访问到关联的透视相机实例。

  • canvases (array): v1.4

    该属性是一个数组，包含了通过加载方法加载到平面中的所有canvas元素。这意味着你可以通过访问这个数组来获取和操作已加载到该平面上的所有HTML canvas元素集合。

  • crossOrigin (string): v1.4

    加载媒体资源时采用的跨域策略

  • cullFace (string): v6.0

    在渲染时应该剔除（即不绘制）多边形的哪一面。其可接受的值包括：
    "back"：默认值，仅绘制平面的正面，背面将被剔除。
    "front"：只绘制平面的背面，而正面会被剔除。
    "none"：不进行剔面操作，即正反两面都会被绘制。

    当您对一个平面沿着 X 轴或 Y 轴进行旋转时，可能会改变原本被认为是“背面”或“正面”的面，因此可能需要更改默认的剔面行为以确保正确显示想要可视化的部分。

    示例： Plane properties and transformations cheat sheet

  • drawCheckMargins (object): v4.0

    在绘制检查计算中添加的附加边距（以像素为单位）。
    正值表示即使在画布外小于该值也会绘制平面，负值表示在画布内大于该值也会隐藏平面。
    如果您弄乱了顶点着色器中的顶点位置，这很有用。 Additional margins to add in the draw check calculations, in pixels.
    Positive value means the plane will be drawn even if outside the canvas by less than the value, negative value means the plane will be hidden if inside the canvas by more than the value.
    Useful if you're messing with the vertices positions in your vertex shader.

    • top float 在比较平面的顶部边缘与画布底部边缘时要应用的边距（默认为0）。
    • right float 在比较平面的右侧边缘与画布左侧边缘时要应用的边距（默认为0）。
    • bottom float 在比较平面的底部边缘与画布顶部边缘时要应用的边距（默认为0）。
    • left float 在比较平面的左侧边缘与画布右侧边缘时要应用的边距（默认为0）。

    示例： Plane properties and transformations cheat sheet

  • gl (WebGL context) v7.0只读

    WebGL 上下文

  • htmlElement (HTML element): v1.0只读

    用于创建平面的HTML元素。如果你想为该平面添加事件监听器时，这个属性很有用。
    你可以在运行时通过调用 resetPlane() 方法来改变平面所使用的HTML元素。

  • images (array): v1.0

    包含了通过加载方法加载到平面中的所有图片资源的数组

  • index (integer): v1.0只读

    当前平面在 Curtains平面数组 中的索引

  • loader (PlaneTextureLoader class object): v7.0只读

    平面内部纹理加载器。

  • quaternion (Quat class object): v6.0updated in v7.0

    一个 Quat 类对象，将 3D 空间中的实际平面旋转表示为四元数。

  • relativeTranslation (Vec3 class object): v1.0updated in v6.1

    一个 Vec3 类对象，其中包含沿 X、Y 和 Z 轴应用于您的平面的附加平移量。

    自 v8.0 版本起，该属性具有响应性，这意味着当您更新 relativeTranslation 向量的一个分量时，平面的平移将自动更新：

    // 将我们的平面沿 X 轴平移 100px
    plane.relativeTranslation.x = 100;

  • renderer (Renderer class object): v7.0只读

    该 Renderer 类对象由我们的 curtains 对象创建。

  • renderOrder (int): v7.3只读

    平面的 renderOrder 值，用于决定多个平面之间的绘制顺序。
    具有较高 renderOrder 数值的平面将会覆盖（渲染在）数值较低的平面之上。
    若要更改此值，可以使用 setRenderOrder() 方法。默认情况下，该值为 0。
    了解更多关于场景中物体如何按照顺序进行绘制的信息，请参阅 场景渲染顺序 相关文档。

  • rotation (Vec3 class object): v1.0

    一个 Vec3 类对象，用于存储应用于你的平面在X、Y和Z轴上的旋转值。

    自 v8.0 版本起，该属性具有响应式特性，这意味着当你更新旋转向量的任意一个分量时，平面的实际旋转将会自动更新。这就意味着你无需手动计算旋转矩阵或四元数来应用新的旋转变化；系统会根据你对Vec3对象组件（如x、y、z）所做的更改自动同步平面的旋转状态。

    // 沿 X 轴将平面旋转 PI / 2
    plane.rotation.x = Math.PI / 2;

  • scale (Vec3 class object): v1.0

    一个 Vec3 类对象，其中包含沿 X 和 Y 轴应用于你的平面的缩放值（Z 轴方向的数值始终为 1）。

    自 v8.0 版本起，该属性具有响应性，这意味着当你更新缩放向量的任一分量时，平面的缩放将会自动更新。

    // 沿 X 轴将我们的平面放大2倍
    plane.scale.x = 2;

  • target (renderTarget): v5.0

    若设置了渲染目标，则平面会渲染到该目标上；
    未设置渲染目标，则平面会直接渲染到画布上，此时该值为null

  • textures (array): v2.0

    包含已创建的所有平面 纹理(Textures) 的数组。

  • transformOrigin (Vec3 class object): v5.0updated in v6.0

    transformOrigin 属性是一个 Vec3 类的对象，用于指定平面在其自身坐标系内变换（旋转、缩放等）的原点位置。这个原点沿着 X、Y 和 Z 轴有具体的坐标值。

    默认情况下设置为 { x: 0.5, y: 0.5, z: 0 }，表示平面中心作为变换原点。其中：
    { x: 0, y: 0, z: 0 } 表示平面左上角；
    { x: 1, y: 1, z: 0 } 表示平面右下角。
    Z 轴上的值尤其有用，它允许你将变换原点设置在平面背后（负值）或平面前方（正值）。这意味着你可以指定的值可以是负数，也可以大于1，这取决于你希望变换操作相对于平面的位置关系。 A Vec3 class object containing your plane transform origin position along X, Y and Z axis.
    { x:0, y:0, z:0 } is the plane top left corner whereas { x:1, y:1, z:0 } is the plane bottom right corner.
    Use the value along the Z axis to set the transformation origin behind (negative value) or in front (positive value) the plane.
    Values could be negative, or greater than 1. Default to { x:0.5, y:0.5, z:0 } (center of the plane).

    示例： Plane properties and transformations cheat sheet

  • type (string): v7.0只读

    类对象类型：“Plane”。

  • uniforms (object): v1.0updated in v7.0

    一个包含您作为参数传递的所有 uniform（统一变量）的对象。
    您可以通过修改其 value 属性来更新 uniform 的值。
    自版本 7.0 起，您可以直接使用 Vec2、Vec3、Mat4 和 Quat 对象作为 uniform 值，而无需使用数组。

  • userData (object): v4.2

    一个空对象，用于将任何附加数据或自定义属性存储到平面中。

  • uuid (string): v6.0只读

    平面的的唯一标识符。

  • videos (array): v1.2

    包含通过加载方法加载到平面中的所有视频的数组。

  • visible (boolean): v4.3

    是否应该绘制您的平面。将此属性设置为 true 或 false 以切换平面的可见性。

    示例： Plane properties and transformations cheat sheet

  • watchScroll (bool): v4.0

    是否在用户滚动时自动更新您的平面位置。

方法

  • addTexture(texture): v7.0

    该函数用于将一个已存在的纹理添加到平面中。

    • texture Texture class object 要添加到此平面的纹理。这个参数等同于 Texture 类中的 addParent() 方法所使用的纹理对象。

    示例： AJAX navigation with plane removal

  • createTexture(params): v2.0updated in v7.0

    此函数用于创建一个新的 Texture并将其添加到我们的 Plane 上。
    如果成功创建，将返回新生成的 Texture 对象。

    • params object，可选 包含纹理参数的对象。如果不指定，将会使用默认的 plane texturesOptions；但至少应提供一个采样器名称。

    返回值：如果创建成功则返回新创建的 Texture

    示例： Slideshow with a displacement shader， Ping pong shading flowmap

  • enableDepthTest(shouldEnableDepthTest): v1.3

    此函数用于开启或关闭指定平面的深度测试功能。

    • shouldEnableDepthTest bool 决定是否为该平面启用深度测试。

  • getBoundingRect(): v3.0

    该方法非常有用，可用于获取平面HTML元素的边界矩形，而无需触发回流（reflow）或布局重绘。请注意，返回的值是相对于 Curtains像素比 的。

    返回值：一个包含平面HTML元素在像素单位下的宽度、高度、顶部、右侧、底部和左侧位置的对象。

    示例： Simple canvas plane， Text planes using canvas， Multiple planes scroll effect ， Post processing multiple passes

  • getWebGLBoundingRect(): v4.0

    该方法用于获取 WebGL 平面元素相对于视口的边界矩形，所有变换（如旋转、缩放和平移）都已考虑在内。它主要用于内部绘制检查函数中（例如视锥体裁剪，frustum culling）。 Useful to get our plane WebGL element bounding rectangle relative to the viewport, with all transformations applied. Used internally in the draw check function (frustum culling).
