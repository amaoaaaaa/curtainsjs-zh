RenderTarget 类

源码

基本使用

该类仅是一个创建帧缓冲对象（FBO）的类。您可以将平面渲染到渲染目标，而不是直接渲染到画布上，然后通过将该渲染目标分配给 ShaderPass 或抓取其内容（即其 Texture纹理），以便在另一个平面上使用。 The RenderTarget class is just a class that creates Frame Buffer Objects (FBO). You can render a plane to a render a target instead of your canvas and then use that render target either by assigning it to a ShaderPass or by grabbing its content (ie its Texture) and use it in another plane.

请注意，渲染目标（因此也包括着色器通道）会禁用 WebGL 上下文的默认抗锯齿功能。如果您使用它们，在初始化上下文时应将Curtains 类的 antialias属性 设置为 false。
之后，您可能希望添加一个额外的抗锯齿通道，可以通过内置的 FXAAPass 类或您自己的 FXAA 或 MSAA 实现来实现这一点。
请参阅 Post processing scrolling wheel with custom transform origin，了解如何添加 FXAAPass 的实例。 Please note that render targets (and therefore shader passes) disable the WebGL context default antialiasing. If you use them, you should set the antialias Curtains property to false when initiating your context.
You might then want to add an extra antialiasing pass, by using the built-in FXAAPass class or your own FXAA or MSAA implementation.
See the Post processing scrolling wheel with custom transform origin for an example of how to add a FXAAPass.

创建

要创建新的渲染目标，只需使用其构造函数：

// “canvas” 是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: "canvas"
});
// 使用我们的curtains对象创建一个新的渲染目标
const renderTarget = new RenderTarget(curtains);
// ...对渲染目标做任何你想做的事

要使用它，您必须使用 Plane 类的 setRenderTarget 将其应用到平面。

在平面中使用 RenderTarget 纹理

// “canvas” 是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: "canvas"
});
// 使用curtains对象创建一个新的渲染目标
const renderTarget = new RenderTarget(curtains);
const planeElement = document.getElementById("my-plane");
// 创建一个平面
const plane = new Plane(curtains, planeElement, {
vertexShaderID: "plane-vs",
fragmentShaderID: "plane-fs",
});
// 将我们的平面添加到我们的帧缓冲区对象中
plane.setRenderTarget(renderTarget);
// 创建一个平面显示我们的渲染目标结果
const renderElement = document.getElementById("render-plane");
const renderPlane = new Plane(curtains, renderElement, {
vertexShaderID: "render-plane-vs",
fragmentShaderID: "render-plane-fs",
});
renderPlane.onReady(function() {
// 创建一个新纹理并将渲染目标纹理复制到其中
// 您将能够在平面着色器中将其用作常规纹理
const renderTexture = renderPlane.createTexture({
sampler: "uRenderTexture",
fromTexture: renderTarget.getTexture(),
});
});

将RenderTarget分配给ShaderPass

您可以选择向一组预定义的平面添加ShaderPass，而不是应用于整个场景。通过将着色器通道与RenderTarget相结合，您能够对所有使用该RenderTarget的平面添加一个额外的渲染通道。这样，您就能针对特定平面集合独立应用特定的后期处理效果或其他图形效果。 You can decide to add a ShaderPass to a defined set of plane instead of your whole scene. By combining a shader pass with a render target, you'll be able to add an additional pass to all the planes that use that render target.

// “canvas” 是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: "canvas"
});
// 使用curtains对象创建一个新的渲染目标
const renderTarget = new RenderTarget(curtains);
const planeElements = document.getElementsByClassName("my-planes");
// 批量创建平面
for(let i = 0; i < planeElements.length; i++) {
const plane = new Plane(curtains, planeElements[i], {
vertexShaderID: "plane-vs",
fragmentShaderID: "plane-fs",
});
// 将我们的平面添加到我们的帧缓冲区对象中
plane.setRenderTarget(renderTarget);
}
// 现在将渲染目标分配给着色器通道
const shaderPassParams = {
vertexShaderID: "my-shader-pass-vs",
fragmentShaderID: "my-shader-pass-fs",
renderTarget: renderTarget, // 这就是我们将渲染目标分配给着色器通道的方式
uniforms: {
time: {
name: "uTime",
type: "1f",
value: 0,
},
},
};
const shaderPass = new ShaderPass(curtains, shaderPassParams);

请查看 Selective shader passes using render targets 示例，了解如何为不同的平面集合应用不同的着色器通道。

参数

以下是完整的参数列表：

  • curtains Curtains class object Curtains 示例对象
  • params object，可选 包含 render target 参数的对象：
    • depth bool，可选 是否创建深度缓冲区（在RenderTarget中处理深度信息）。默认为false。
    • clear bool，可选 在渲染目标被绘制之前，是否应清除其内容。在处理乒乓（ping-pong）着色等技术时，应将其设置为false。默认情况下为true。
    • maxWidth float，可选 渲染目标的最大宽度。
    • maxHeight float，可选 渲染目标的最大高度。
    • minWidth float，可选 渲染目标的最小宽度。默认为 1024。
    • minHeight float，可选 渲染目标的最小高度。默认为 1024。
    • texturesOptions object，可选 应用于渲染目标纹理的默认选项。请参阅Texture类参数。

属性

  • index(integer): v5.0 只读

    此属性表示渲染目标在 renderTargets 数组中的索引位置。

  • gl (WebGL context) v7.0只读

    WebGL 上下文

  • renderer (Renderer class object): v7.0只读

    由我们的curtains对象创建的渲染器。

  • textures(array): v5.0

    一个长度为1的数组，其中包含渲染目标所对应的 Texture 对象。

  • type (string): v7.0只读

    类对象类型: "RenderTarget"。

  • userData (object): v5.0

    一个空的对象，用于在渲染目标中存储任何额外的数据或自定义属性。

  • uuid (string): v6.0只读

    渲染目标的唯一标识符。

方法

  • getTexture(): v7.0

    返回渲染目标的纹理，以便后续使用。

    返回值：渲染目标对应的纹理对象。

  • remove(): v7.0

    此函数会从Curtains对象及其渲染器中移除该渲染目标，并删除其关联的纹理。同时，它还会更新所有其他渲染目标的索引。
