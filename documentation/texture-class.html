基本使用

创建

有多种方法可以创建纹理。
最简单的一种方法是简单地调用它的构造函数：

// “canvas”是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: "canvas"
});
// 使用我们的curtains对象创建一个新纹理
// 使用“uTexture”作为采样器名称
const texture = new Texture(curtains, {
sampler: "uTexture"
});
// ...您可以稍后使用 addParent() 方法将其添加到平面或着色器通道

您也可以通过首先创建一个 TextureLoader ，然后使用它来加载HTML媒体元素来创建纹理：

// "canvas" 是我们HTML容器元素的ID
const curtains = new Curtains({
container: "canvas"
});
// 创建一个新的纹理加载器
const loader = new TextureLoader(curtains);
// 使用加载器加载图片
const image = new Image();
image.crossOrigin = "anonymous";
image.src = "path/to/my-image.jpg";
let newTexture;
loader.loadImage(image, {
// 纹理选项（这里我们只设置其采样器名称）
sampler: "uTexture"
}, (texture) => {
// 纹理已成功创建
newTexture = texture;
// ...您可以使用addParent()方法将其添加到平面或着色器通道中
}, (image, error) => {
// 在加载图片时出现错误
});

另一种创建纹理的方式是使用 createTexture 方法，该方法属于 Plane 或 ShaderPass 类对象：

// "canvas" 是我们HTML容器元素的ID
const curtains = new Curtains({
container: "canvas"
});
// 获取页面上用于平面的HTML元素
const planeElement = document.getElementById("my-plane");
// 创建一个新的平面对象
const plane = new Plane(curtains, planeElement);
// 使用平面的createTexture方法创建新纹理，并将采样器名称设为"uTexture"
const myTexture = plane.createTexture({
sampler: "uTexture"
});

每当调用平面加载方法时，纹理也会自动创建。另外，如果平面的autoloadSources初始参数设置为true（默认情况下就是true），则在初始化阶段，对于平面元素下每一个图片、canvas或视频子元素，都会自动生成相应的纹理。

纹理选项

在创建纹理时，你可以传递一些选项来定义纹理的各种属性，例如放大和缩小滤镜、各向异性过滤、纹理环绕方式或预乘Alpha等。
想要了解所有可选参数，请参阅完整的参数列表。

// "canvas" 是我们HTML容器元素的ID
const curtains = new Curtains({
container: "canvas"
});
// 使用我们的curtains对象创建一个新纹理
// 使用“uTexture”作为采样器名称并设置一些选项
const texture = new Texture(curtains, {
// 将预乘Alpha设置为true，设置最小过滤器（minFilter）、各向异性（anisotropy），并使用半精度浮点数纹理
sampler: "uTexture",
premultiplyAlpha: true,
minFilter: curtains.gl.LINEAR_MIPMAP_NEAREST,
anisotropy: 16,
floatingPoint: "half-float"
});
// ...您可以稍后使用 addParent() 方法将其添加到平面或着色器通道

内置统一变量名称

默认情况下，发送至着色器的纹理采样器和矩阵的统一变量名将根据它们在平面元素内部媒体元素的顺序设置，遵循以下规则： By default the textures sampler and matrix uniform names sent to your shaders will be set upon their media element order inside your plane element, following this rule:

纹理采样器：`uSampler{index}`
纹理矩阵：`uTextureMatrix{index}` `uSampler{index}` for texture samplers
`uTextureMatrix{index}` for texture matrices

HTML
<!-- div 用于创建我们的平面 -->
<div id="my-plane">
<!-- 这些图片将被作为平面的纹理 -->
<img src="path/to/my-image-1.jpg" crossorigin="" />
<img src="path/to/my-image-2.jpg" crossorigin="" />
</div>
Javascript
// “canvas”是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: "canvas"
});
const planeElement = document.getElementById("my-plane");
const plane = new Plane(curtains, planeElement);
Shaders
// 在顶点着色器中使用：
uniform mat4 uTextureMatrix0; // 纹理矩阵，对应于 my-image-1.jpg
uniform mat4 uTextureMatrix1; // 纹理矩阵，对应于 my-image-2.jpg
...
// 在片段着色器中使用：
uniform sampler2D uSampler0; // 采样器，与 my-image-1.jpg 绑定
uniform sampler2D uSampler1; // 采样器，与 my-image-2.jpg 绑定

使用自定义统一变量名称

您可以通过为媒体HTML元素指定data-sampler属性来自定义统一变量名称。此时，统一变量名将遵循以下规则：

纹理采样器：`{data-sampler}`
纹理矩阵：`{data-sampler}Matrix`

HTML
<!-- 用于创建平面的div -->
<div id="my-plane">
<!-- 将作为平面纹理使用的图片 -->
<img src="path/to/my-image-1.jpg" crossorigin="" data-sampler="firstTexture" />
<img src="path/to/my-image-2.jpg" crossorigin="" data-sampler="secondTex" />
</div>
Shaders
// 顶点着色器（Vertex Shader）：
uniform mat4 firstTextureMatrix; // 对应于 my-image-1.jpg 的纹理矩阵
uniform mat4 secondTexMatrix; // 对应于 my-image-2.jpg 的纹理矩阵
...
// 片段着色器（Fragment Shader）：
uniform sampler2D firstTexture; // 绑定到 my-image-1.jpg
uniform sampler2D secondTex; // 绑定到 my-image-2.jpg

新建的纹理在您为其设置源之前，将只会显示黑色像素。您需要在您的场景中选取一个加载好的图片、画布或视频资源作为该纹理的数据源。

使用纹理矩阵

着色器使用纹理坐标（或UV坐标）将纹理映射到平面。
默认情况下，库设置了一个任意的aTextureCoord属性，您可以在着色器中使用它： Shaders use texture coordinates (or UVs) to map a texture to the plane.
By default the library sets an arbitrary aTextureCoord attributes that you could use in your shaders:

<!-- 顶点着色器 -->
<script id="plane-vs" type="x-shader/x-vertex">
precision mediump float;
// 这些是库设置的强制属性
attribute vec3 aVertexPosition;
attribute vec2 aTextureCoord;
// 这些是库设置的必要 uniform 变量，它们包含我们的模型视图矩阵和投影矩阵
uniform mat4 uMVMatrix;
uniform mat4 uPMatrix;
// 将顶点坐标和纹理坐标传递给片段着色器
varying vec3 vVertexPosition;
varying vec2 vTextureCoord;
void main() {
gl_Position = uPMatrix * uMVMatrix * vec4(aVertexPosition, 1.0);
// 设置变量
// 在片段着色器中使用我们的 aTextureCoord 属性作为纹理坐标
vTextureCoord = aTextureCoord;
vVertexPosition = aVertexPosition;
}
</script>
<!-- 片段着色器 -->
<script id="plane-fs" type="x-shader/x-fragment">
precision mediump float;
// 获取我们的变量
varying vec3 vVertexPosition;
varying vec2 vTextureCoord;
// 我们的纹理采样器（默认采样器名称）
uniform sampler2D uSampler0;
void main() {
// 用原始纹理坐标映射我们的纹理
gl_FragColor = texture2D(uSampler0, vTextureCoord);
}
</script>

您可以使用它将纹理映射到平面，但如果平面和平面媒体的宽高比不同，最终会得到一个扭曲的纹理。

幸运的是，有一种内置解决方案可以避免纹理被扭曲地显示：纹理矩阵。每当您将源设置为纹理（或每次平面被调整大小时），其纹理矩阵会自动计算，并作为 uniform 发送给您的着色器。以上述示例为例，以下是使用它的方法： Fortunately, there's a built-in solution to display your textures without them being distorted: texture matrices. Each time you set a source to a texture (or each time a plane is resized), its texture matrix is automatically computed and sent to your shader as a uniform. With the same example as above, here's how to use it:

<!-- 顶点着色器 -->
<script id="plane-vs" type="x-shader/x-vertex">
precision mediump float;
// 这些是库设置的强制属性
attribute vec3 aVertexPosition;
attribute vec2 aTextureCoord;
// 这些是库设置的必要 uniform 变量，它们包含了模型视图矩阵和投影矩阵
uniform mat4 uMVMatrix;
uniform mat4 uPMatrix;
// 我们的纹理矩阵，用于处理图像的完整覆盖
uniform mat4 uTextureMatrix0;
// 将顶点坐标和纹理坐标传递给片段着色器
varying vec3 vVertexPosition;
varying vec2 vTextureCoord;
void main() {
gl_Position = uPMatrix * uMVMatrix * vec4(aVertexPosition, 1.0);
// 设置变量
// 在这里，我们使用纹理矩阵来计算准确的纹理坐标
vTextureCoord = (uTextureMatrix0 * vec4(aTextureCoord, 0.0, 1.0)).xy;
vVertexPosition = aVertexPosition;
}
</script>
<!-- 片段着色器 -->
<script id="plane-fs" type="x-shader/x-fragment">
precision mediump float;
// 获取我们的变量
varying vec3 vVertexPosition;
varying vec2 vTextureCoord;
// 我们的纹理采样器（默认采样器名称）
uniform sampler2D uSampler0;
void main() {
// 使用纹理矩阵映射纹理坐标
gl_FragColor = texture2D(uSampler0, vTextureCoord);
}
</script>

纹理矩阵 uniform 变量的命名规则与采样器 uniform 变量相同：
如果您没有指定名称，它将按照 "uTextureMatrix" + 您纹理的索引进行命名。
如果您指定了采样器名称，则其命名为 sampler 名称 + "Matrix"。 The texture matrix uniform name will follow the same rule as the sampler uniform name:
If you did not specify any name, it will be named "uTextureMatrix" + your texture index.
If you did specify a sampler name, it will be named samplerName + "Matrix".

参数

以下是纹理的完整参数列表：

  • curtains Curtains class object Curtains 实例对象
  • params object，可选 包含纹理参数的对象：
    • sampler string，可选 在您的采样器和纹理矩阵 uniform 中使用的采样器名称。
    • fromTexture Texture class object，可选 将传入的纹理复制到此纹理中
    • premultiplyAlpha bool，可选 是否应对此纹理使用预乘 alpha。默认为 false。
    • floatingPoint string，可选 是否应对此纹理使用浮点数（如果可用，取决于WebGL上下文和扩展）。接受的值为 "none"、"half-float" 和 "float"。默认为 "none"。
    • anisotropy integer，可选 纹理各向异性值（参见 各向异性过滤 的解释）。通常范围从1到16，具体取决于WebGL上下文和可用扩展。默认为1。
    • generateMipmap bool，可选 是否为该纹理生成 mipmaps（参见 mip maps 的解释）。对于图像纹理，默认设置为 true。请注意，视频和画布纹理将永远不会生成 mipmaps，因为它们频繁上传。
    • minFilter GLenum，可选 纹理缩小过滤器。根据 WebGL 上下文和源图像的大小，使用 可能的GLenum值 之一。 一些 WebGL1 的过滤器需要源图像尺寸为 2 的幂次方。默认设置为 gl.LINEAR。
    • magFilter GLenum，可选 纹理放大过滤器。根据 WebGL 上下文和源图像的大小，使用 可能的GLenum值 之一。 一些 WebGL1 的过滤器需要源图像尺寸为 2 的幂次方。默认设置为 gl.LINEAR。
    • wrapS GLenum，可选 沿 X 轴的纹理环绕方式。根据 WebGL 上下文和源图像尺寸，选择 合适的 GLenum可能值 之一。 一些 WebGL1 的环绕模式可能需要源纹理尺寸为 2 的幂次方。默认设置为 gl.CLAMP_TO_EDGE。
    • wrapT GLenum，可选 沿 Y 轴的纹理环绕方式。根据 WebGL 上下文和源图像尺寸，选择 合适的 GLenum可能值 之一。 一些 WebGL1 的环绕模式可能需要源纹理尺寸为 2 的幂次方。默认设置为 gl.CLAMP_TO_EDGE。

属性

  • gl (WebGL context) v7.0只读

    WebGL 上下文

  • index (integer): v2.0 只读

    该纹理在 Plane 或 ShaderPass 纹理数组中的索引位置。
    由于 ShaderPass 在初始化时会自动创建一个纹理，如果你稍后向其中添加纹理，那么新添加的纹理将具有索引值1。

  • offset (Vec2 class object): v7.3

    一个包含在 x 和 y 轴上应用于纹理的偏移量的对象。仅当您在着色器中使用纹理矩阵映射纹理时才会应用此偏移量。

  • parameters (object): v7.0只读

    实际使用的纹理参数值：anisotropy，generateMipmap，minFilter，magFilter，wrapS，wrapT

  • renderer (Renderer class object): v7.0只读

    由curtains对象创建的渲染器。

  • scale (Vec2 class object): v2.0

    一个包含应用于纹理在x轴和y轴上的缩放比例的对象。该属性仅在您使用纹理矩阵在着色器中映射纹理时生效。

  • shouldUpdate (boolean): v2.0

    这取决于您的纹理 sourceType属性。
    默认情况下，canvas 纹理会随着每次 requestAnimationFrame 调用而更新，video 纹理会当下一帧可用时更新，而 image 纹理则永远不会被更新。
    如果您希望停止更新 canvas 或 video 纹理以提高性能，请将此标志设置为 false。

    示例： Text planes using canvas

  • source (HTML element): v2.0

    用于纹理的源，可以是图像、画布或视频HTML元素。通过它您可以安全地访问该元素的任何属性或方法，例如对视频调用 play() 或 pause() 方法。

  • sourceType (string): v7.0只读

    表示纹理的类型。可能的值包括 “empty”、“image”、“canvas”、“video” 或 “fboTexture”。
    其中，“fboTexture”是一种 RenderTarget 纹理，它是用于绘制您的平面或场景的目标。

  • type (string): v7.0只读

    类的类型：“Texture”

  • userData (object): v5.0

    一个空对象，用于在纹理中存储任何附加数据或自定义属性。

  • uuid (string): v6.0只读

    该纹理的唯一标识符。

方法

  • addParent(parent): v7.0

    将您的纹理添加到一个 平面（Plane） 或一个 着色器通道（ShaderPass） 中，以便能够显示该纹理。

    • parent Plane class object or ShaderPass class object 应使用该纹理的WebGL对象。

    示例： AJAX navigation with plane removal

  • copy(texture): v7.0

    将一个已存在的纹理复制到当前纹理中。

    • texture Texture class object 需要复制的目标纹理

    示例： Ping pong shading flowmap

  • hasParent(): v7.0

    判断当前纹理是否已关联父对象。

    返回值：如果该纹理已关联父对象，则返回true；否则返回false。

    示例： AJAX navigation with plane removal

  • needUpdate(): v4.1

    强制纹理在下一次绘制调用期间进行更新，即使其 shouldUpdate 属性被设置为false。如果你希望仅在某次情况下强制更新canvas纹理，可以使用此方法。

    示例： Text planes using canvas

  • resize(): v4.2

    强制更新你的纹理矩阵。当你手动调整源对象（通常是一个canvas）大小时，这个方法特别有用。

    示例： Text planes using canvas

  • setAnisotropy(anisotropy): v7.0

    设置纹理的各向异性程度值（关于各向异性过滤的详细解释）。

    • anistropy integer 需要应用的各向异性程度值（通常在 1 - 16 之间）

    示例： Plane properties and transformations cheat sheet

  • setMagFilter(magFilter): v7.0

    设置纹理的放大过滤器。

    • magFilter GLenum 要应用的放大过滤器。请根据WebGL上下文和源图像尺寸，使用 可能的GLenum值 中的一个（某些WebGL1的过滤器要求源图像尺寸为2的幂次方大小）。

  • setMinFilter(minFilter): v7.0

    设置纹理的缩小过滤器。

    • minFilter GLenum 要应用的缩小过滤器。请根据WebGL上下文和源图像尺寸，使用 可能的GLenum值 中的一个（某些WebGL1的过滤器要求源图像尺寸为2的幂次方大小）。

    示例： Slideshow with a displacement shader

  • setOffset(offset): v7.3

    设置纹理偏移量。仅在您在着色器中使用纹理矩阵映射纹理时应用。

    • scale Vec2 class object 沿 X 和 Y 轴应用的偏移量（沿 X 轴为 0.5 表示宽度的一半）。

    示例： GSAP click to fullscreen gallery

  • setScale(scale): v2.0updated in v7.0

    设置纹理的新缩放比例。仅在您使用纹理矩阵在着色器中映射纹理时生效。

    • scale Vec2 class object 沿 X 轴和 Y 轴要设置的缩放比例。

    示例： Multiple planes scroll effect : rotation, scale and parallax， Post processing multiple passes ， GSAP click to fullscreen gallery

  • setSource(sourceElement): v2.0

    使用此方法为您的纹理设置源。该源必须已经加载到您的平面中，否则请使用 Plane 或 ShaderPass 类的加载方法之一。

    • sourceElement either an image, canvas or video HTML element 要应用于纹理的源，如图片、画布或视频等HTML元素，它必须是由您的平面已加载完成的元素。

    示例： Slideshow with a displacement shader

  • setWrapS(wrapS): v7.0

    设置纹理的 wrapS 值。

    • wrapS GLenum 沿着X轴应用到纹理上的环绕模式。使用 可能的GLenum值 之一，具体取决于WebGL上下文和源图像的大小（某些 WebGL1 中的环绕模式需要源图像尺寸为2的幂次方）。

  • setWrapT(wrapT): v7.0

    设置纹理的 wrapT 值。

    • wrapT GLenum 沿着Y轴应用到纹理上的环绕模式。使用 GLenum可能值 之一，具体取决于 WebGL 上下文和源图像尺寸（某些 WebGL1 的环绕模式需要源图像的高度为2的幂次方）。

事件

  • onSourceLoaded(callback): v7.0

    当纹理源首次加载完成时，该函数将被调用。

    • callback function 要执行的回调函数。

    返回值：你的 Texture 对象，以便于进行链式调用。

  • onSourceUploaded(callback): v7.0

    当您的纹理源首次上传到GPU时，该函数将被调用。

    • callback function 要执行的回调函数。

    返回值：你的 Texture 对象，以便于进行链式调用。

    示例： AJAX navigation with plane removal， GSAP click to fullscreen gallery
