基本使用

您必须首先创建一个 Curtains 对象来处理包含所有平面的场景。它还将创建 WebGL 上下文、附加画布并处理 requestAnimationFrame 循环。您只需传递将包裹画布的 HTML 元素（或其 ID）： You will have to create a Curtains object first that will handle the scene containing all your planes. It will also create the WebGL context, append the canvas and handle the requestAnimationFrame loop. You just have to pass the HTML element (or its ID) that will wrap the canvas :

// “canvas”是我们的 HTML 容器元素的 ID
const curtains = new Curtains({
container: document.getElementById("canvas") // 我们可以直接传递DOM
});

还有其他选项可以作为参数传递。例如，您可以指定是否要在生产模式下使用该库： There are other options that you can pass as parameters. For example, you can specify whether you want to use the library in production mode:

const curtains = new Curtains({
container: "canvas", // 这里传递的是容器的ID名称
production: true // 在“生产”模式下使用库
});

参数

以下是所有参数的完整列表：

  • container HTML DOM元素或元素ID的字符串，可选 将包裹画布的 HTML 元素或 HTML 元素的 ID。如果未指定，则在使用有效的 HTML 元素或 ID 调用 setContainer() 之前，不会设置 WebGL 上下文。
  • alpha boolean，可选 WebGL 上下文是否应处理透明度。默认为 true。
  • antialias boolean，可选 WebGL 上下文是否使用抗锯齿。默认为 true。使用渲染目标时，WebGL 会禁用抗锯齿功能，因此您可以安全地将其设置为 false 以提高性能。
  • premultipliedAlpha boolean，可选 WebGL 上下文是否应处理预乘 alpha。默认为 false。
  • depth boolean，可选 WebGL 上下文是否应该处理深度。默认为 true。
  • preserveDrawingBuffer boolean，可选 WebGL 上下文是否应保留绘图缓冲区。默认为 false。
  • failIfMajorPerformanceCaveat boolean，可选 如果出现重大性能问题，WebGL 上下文创建是否会失败。默认为 true。
  • stencil boolean，可选 WebGL 上下文是否应该处理模板。默认为 false。
  • autoRender boolean，可选 是否创建 requestAnimationFrame 循环来渲染场景。如果您想使用 render() 方法自己处理此问题，请将其设置为 false。默认为 true。
  • autoResize boolean，可选 是否侦听窗口调整大小事件并实际调整场景大小。如果您想使用 resize() 方法自行处理此问题，请将其设置为 false。默认为 true。
  • pixelRatio float，可选 定义像素比率值。使用它来限制 init 以提高性能。默认为 window.devicePixelRatio。
  • renderingScale float，可选（最小值：0.25，最大值：1） 用它来缩小渲染画布的比例。可能会提高性能，但会降低质量。默认为 1。
  • watchScroll boolean，可选 是否监听窗口滚动事件。如果你想自己处理这个问题，请将其设置为 false。默认为 true。
  • production boolean，可选 是否抛出有用的控制台警告。默认为 false。

属性

  • alpha (boolean) v5.0只读

    是否处理透明度

  • antialias (boolean) v6.1只读

    是否使用抗锯齿

  • canvas (HTML canvas 元素) v7.0

    正在创建的 WebGL 场景的 canvas 元素

  • container (HTML 元素) v1.0

    将容纳我们的 WebGL canvas 的容器。canvas 的大小将根据该容器进行调整。出于性能原因，容器越小越好。大多数时候，您会将其设置为全屏。

  • depth (boolean) v7.0只读

    WebGL 上下文是否处理深度。

  • failIfMajorPerformanceCaveat (boolean) v7.0只读

    如果出现重大性能问题，WebGL 上下文创建是否会失败。

  • gl (WebGL 上下文) v5.0只读

    创建的 WebGL 上下文

  • pixelRatio (float) v1.0 只读

    当前设备像素比。使用 setPixelRatio() 方法在运行时更改此值。

  • planes (array) v1.0

    包含当前所有 Plane 元素的数组。

  • premultipliedAlpha (boolean) v5.0 只读

    WebGL 上下文是否处理预乘 Alpha。

  • preserveDrawingBuffer (boolean) v7.0只读

    WebGL 上下文是否保留绘图缓冲区。

  • production (boolean) v7.0

    是否生产模式，如果设置为 true，将删除库显示的所有有用警告。

  • renderTargets (array) v5.0

    包含所有当前 RenderTarget 元素（包括着色器通道使用的元素）的数组。

  • shaderPasses (array) v3.0

    包含所有当前 着色器通道(ShaderPass) 元素的数组。

  • stencil (boolean) v7.0只读

    WebGL 上下文是否处理模板。

方法

  • clear(): v7.3

    清除 WebGL 上下文颜色和深度缓冲区。

  • clearColor(): v7.3

    清除 WebGL 上下文颜色缓冲区。

  • clearDepth(): v7.3

    清除 WebGL 上下文深度缓冲区。

  • disableDrawing(): v1.8

    此函数将防止再次绘制场景（将其置于暂停状态）。当绘图被禁用时，您将无法更新着色器 uniforms。如果您有静态场景，这对于提高性能很有用。

    示例：Plane properties and transformations cheat sheet， Slideshow with a displacement shader， Asynchronous textures loading

  • dispose(): v1.0

    该函数将取消 requestAnimationFrame 循环，删除所有平面并删除 WebGL 上下文。

  • enableDrawing(): v1.8

    如果您通过 disableDrawing() 暂停场景绘制，此函数将重新启用场景绘制。例如，如果您想在用户手势发生时再次开始绘制场景，这可能会很有用。

    示例：Slideshow with a displacement shader， Asynchronous textures loading

  • getBoundingRect(): v3.0

    获取 container 元素边界矩形，而不会导致回流/布局重绘。请小心，因为这些值与您的 PixelRatio 值相关。

    返回：一个包含 container 元素宽度、高度、顶部和左侧位置的对象。

    示例：Multiple planes scroll effect， Post processing multiple passes

  • getScrollDeltas(): v4.0

    获取沿 X 轴和 Y 轴的最后滚动增量值。如果滚动监视处于活动状态，则在调用 updateScrollValues() 后或在 onScroll() 事件内调用它。

    返回：包含沿 X 轴和 Y 轴的最后滚动增量值的对象。

    示例：Multiple planes， Multiple planes scroll effect : rotation, scale and parallax， Post processing displacement effect， Post processing multiple passes

  • getScrollValues(): v4.0

    获取沿 X 轴和 Y 轴的当前滚动值。如果滚动监视处于活动状态，则在调用 updateScrollValues() 后或在 onScroll() 事件内调用它。

    返回： 包含沿 X 轴和 Y 轴的当前滚动值的对象。

    示例：Multiple planes， Multiple planes scroll effect : rotation, scale and parallax， Post processing displacement effect， Post processing multiple passes

  • isWebGL2(): v7.3

    检查创建的 WebGL 上下文是否使用 WebGL2。

    返回：一个boolean，指示上下文是否使用 WebGL2。

  • lerp(start, end, amount): v7.0

    计算起始值和结束值之间指定量的线性插值（LERP是Linear Interpolation的缩写，即线性插值） Calculates a linear interpolation of the amount specified between the start and end values

    • start float 起始值
    • end float 结束值
    • amount float 插值系数

    返回： 线性插值结果（float）。

    示例： Simple plane， Multiple planes， Multiple planes scroll effect : rotation, scale and parallax， Simple video plane， Simple canvas plane， Post processing displacement effect， Post processing multiple passes， Post processing scrolling wheel with custom transform origin， Ping pong shading / FBOs swapping flowmap， Selective shader passes using render targets， Multiple planes scroll effect with Locomotive Scroll

  • needRender(): v1.8

    重新启用一帧的场景绘制。如果您想在绘图被禁用的情况下更新uniforms，这很有用。

    示例：Slideshow with a displacement shader， Asynchronous textures loading

  • nextRender(callback, keep): v7.0updated in v8.0

    在下一次渲染调用时执行回调函数。

    • callback function 回调函数
    • keep boolean 是否保持执行回调函数。如果true，则会在每次渲染时都调用回调函数；如果false，则只会在下一次渲染时调用回调函数。默认为false。

    返回：一个回调队列项。如果您需要在满足特定条件后将该队列项的 keep 属性从 true 更改为 false（实现类似 clearInterval 的效果），那么这个返回值将会非常有用。

    示例： Plane properties and transformations cheat sheet， AJAX navigation with plane removal

  • render(): v4.0

    此函数会在启用绘图时渲染您的场景。这对于您正在使用自定义的 requestAnimationFrame 动画循环特别有用。

  • resize(): v2.0 触发重排（reflow）

    此函数会调整您的场景及其所有平面的大小。它会在每次窗口resize事件时自动内部调用。如果您在不改变窗口尺寸的情况下动态更改容器尺寸，应当手动调用此函数。

  • restoreContext(): v2.0

    此函数将尝试恢复WebGL上下文。在之前失去上下文后，请使用它来尝试恢复。

    示例： examples.

  • setContainer(container): v6.2

    设置Curtains container，我们将在此container中追加canvas元素，并开始实例化Curtains类和WebGL上下文。
    如果在初始参数对象中指定了一个container，则此操作会自动内部调用；若初始参数中未提供container，则可以在稍后调用此方法。

    • container HTML Element or string HTML 元素或元素ID，用于将canvas元素追加到指定的容器中。

  • setPixelRatio(pixelRatio, triggerCallback): v2.0 触发重排（reflow）

    此函数将设置一个新的像素比，并相应地调整所有元素的大小。
    初始化时，默认的像素比值为实际设备像素比。若要限制像素比以提升性能，可以使用此方法。

    • pixelRatio float 要设置的新像素比
    • triggerCallback boolean，可选 是否触发 onAfterResize 回调，默认 false。

  • updateScrollValues(xOffset, yOffset): v4.0

    更新滚动值。在使用 updateScrollPosition() 更新平面位置的同时滚动时，先调用此方法。
    如果滚动监听处于激活状态，则会自动内部调用此函数。

    • xOffset float 沿 X 轴滚动值
    • yOffset float 沿 Y 轴滚动值

    示例： Multiple planes scroll effect with Locomotive scroll

事件

  • onAfterResize(callback): v3.0

    每当窗口大小调整完成后，此函数将被调用。

    • callback function 需要执行的回调函数。

    返回值：你的Curtains对象，这样可以支持链式调用。

  • onContextLost(callback): v2.0

    此函数会在WebGL上下文丢失时每次被调用。

    • callback function 需要执行的回调函数。

    返回值：你的Curtains对象，这样可以支持链式调用。

    示例： examples

  • onContextRestored(callback): v2.0

    此函数会在WebGL上下文成功恢复时每次被调用。

    • callback function 需要执行的回调函数

    返回值：你的Curtains对象，这样可以支持链式调用。

  • onError(callback): v1.7

    此函数会在初始化过程中发生错误，或者无法成功创建WebGL上下文时被调用。

    • callback function 需要执行的回调函数

    返回值：你的Curtains对象，这样可以支持链式调用。

    示例： examples.

  • onRender(callback): v2.0

    此函数在每次调用 requestAnimationFrame 时被调用一次。如果你需要更新性能统计信息或使用补间动画引擎，这个函数会非常有用。

    • callback function 需要执行的回调函数

    返回值：你的Curtains对象，这样可以支持链式调用。

    示例： Multiple planes， Multiple planes scroll effect : rotation, scale and parallax

  • onScroll(callback): v4.0

    这个函数在滚动事件触发并且滚动监听处于激活状态时，每次都会被调用。

    • callback function 需要执行的回调函数

    返回值：你的Curtains对象，这样可以支持链式调用。

    示例： Multiple planes， Multiple planes scroll effect : rotation, scale and parallax， Post processing displacement effect ， Post processing multiple passes

  • onSuccess(callback): v8.1

    当WebGL 成功创建时将调用此函数。

    • callback function 需要执行的回调函数

    返回值：你的Curtains对象，这样可以支持链式调用。
